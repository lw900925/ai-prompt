# RIPER 记忆库

此目录包含用于RIPER工作流程管理的持久化记忆系统。

## 目录结构

```text
memory-bank/
├── [分支名称]/             # 分支特定的记忆存储
│   ├── plans/             # 技术规范与实施计划
│   ├── reviews/           # 代码评审会话与反馈
│   └── sessions/          # 会话历史与上下文记录
└── README.md              # 本说明文档
```

## 分支组织
记忆库会根据需要自动创建分支特定的目录：

### main/
主开发分支的默认记忆存储。

### [功能分支]/
每个功能分支都拥有其独立的记忆空间。

## 使用方式

记忆库由RIPER智能体自动管理：

- 计划子模式：在 plans/ 目录中创建并存储技术规范
- 评审模式：在 reviews/ 目录中存储代码评审会话和反馈
- 记忆命令：在 sessions/ 目录中保存/检索会话上下文

## 文件命名约定

- **计划文件**：[分支名]-[日期]-[功能描述].md
- **评审文件**：[分支名]-[日期]-[评审范围].md
- **会话文件**：[日期]-[主题].md

## 记忆管理

- 文件由RIPER智能体自动创建
- 分支特定的隔离防止交叉污染
- 持久化存储支持跨会话的上下文连续性

## 访问控制

- 读取权限：所有RIPER智能体和模式
- 写入权限：受模式和分支上下文限制
- 允许手动编辑，但建议使用RIPER命令以确保一致性
