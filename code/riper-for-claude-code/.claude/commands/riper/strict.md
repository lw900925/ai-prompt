## 协议规则

1. **模式声明**: 每个响应都以 `[MODE: X]` 或 `[NO MODE]` 开头
2. **模式切换**: 只有您可以授权模式变更
3. **模式限制**: 每个模式都有特定的允许操作
4. **违规处理**: 任何超出当前模式的操作都会触发阻断警告

## 可用命令 (RIPER架构)

要进入一个模式，请使用以下命令之一:
- `/riper:research` - 进入研究子模式 (只读)
- `/riper:innovate` - 进入创新子模式 (头脑风暴)
- `/riper:plan` - 进入计划子模式 (制定规范)
- `/riper:execute` - 进入执行子模式 (需要已批准的计划)
- `/riper:review` - 进入评审模式 (验证)

注意: RIPER使用3个整合后的智能体及其子模式以提升性能

或者明确说出:
- "进入RESEARCH模式"
- "进入INNOVATE模式"
- "进入PLAN模式"
- "批准PLAN并进入EXECUTE模式"
- "进入REVIEW模式"
